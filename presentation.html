01 — Pamje e Përgjithshme

Çfarë është Mansio?

Mansio është një sistem i plotë menaxhimi hoteli i ndërtuar si aplikacion web full-stack. Sistemi lejon:

  • Menaxhimin e stafit dhe mysafirëve
  • Kontrollin e dhomave dhe llojeve të dhomave
  • Sistemin e rezervimeve (në zhvillim)
  • Dashboard administrativ me statistika
  • Kontroll të aksesit bazuar në role (RBAC)

Rolet e Sistemit

  • ADMIN — Akses i plotë
  • MANAGER — Menaxhim operacional
  • RECEPTIONIST — Check-in/out, mysafirë
  • HOUSEKEEPING — Statusi i dhomave
  • GUEST — Profili personal

Arkitektura

Monorepo me backend/ dhe frontend/ si paketa të pavarura. Backend REST API + Frontend SPA të ndarë komplet.

05 — Autentikimi

Sistemi JWT me Refresh Token Rotation

Login Flow

1. Verifiko email → ngarko User + roles
2. Kontrollo status (LOCKED/DISABLED/PENDING → 403)
3. bcrypt.compare(password, hash)
→ Dështim: rrit access_failed_count
→ 5 dështime: status = LOCKED
4. Sukses: reset access_failed_count = 0
5. Gjenero accessToken (15 min)
6. Gjenero refreshToken (7 ditë)
7. Ruaj refreshToken në DB (RefreshTokens)

Refresh Token Rotation

Çdo herë që përdoruesi kërkon access token të ri:

  • Verifiko JWT nënshkrimin
  • Kontrollo DB: ekziston + jo revokuar + jo skaduar
  • Kontrollo që user është ende ACTIVE
  • Revoko tokenin e vjetër (revoked = true)
  • Lësho çift të ri access + refresh

Vendim Arkitekturor: Stateful Auth

authMiddleware ngarkon User-in nga DB çdo kërkesë — jo vetëm dekodon JWT.

Kjo garanton që nëse një llogari LOCKED-ohet, efekti është i menjëhershëm — nuk pret skadimin e access token (15 min).

authMiddleware.ts

  • Ekstrakto Bearer <token>
  • Verifiko JWT me JWT_ACCESS_SECRET
  • Ngarko User + roles nga DB
  • LOCKED/DISABLED/PENDING → 403
  • Shtoje req.user për controllers

roleMiddleware.ts

  • Factory: roleMiddleware(['ADMIN'])
  • Lexon req.user.user_roles
  • Kontrollo ndeshje me allowed roles
  • 401 nëse nuk ka user
  • 403 nëse role e gabuar

validateRequestMiddleware.ts

  • Generic: <T>(schema, source)
  • Source: body | params | query
  • 400 me details nëse invalid
  • Zëvendëson req.body me të dhëna të pastruara
  • Controllers marrin tipe TypeScript të sakta

Siguria kundër SQL Injection: Çdo param :id kalon nëpër numericStringSchema = z.string().regex(/^[0-9]+$/) — parandalon injeksion me string jo-numerik.

10 — Vendime Arkitekturore

Pse u Zgjodh Kjo Arkitekturë?

Logjikë e pastër e shtresave

Routes → Middleware → Controller → Service → Prisma. Asnjë logjikë biznesi jashtë service layer. Lejon testim të izoluar.

Zod si single source of truth

Skema Zod gjeneron automatikisht TypeScript types (z.infer<>). Asnjë duplikim — validimi dhe tipat janë gjithmonë sinkronizuar.

safeUserSelect — Parandalim ekspozimi

Konstantë e vetme që eksklud password_hash, lockout_enabled nga çdo response. Nuk mund të harrohet nga ndonjë developer.

Stateful JWT vs Stateless

Secilia kërkesë e mbrojtur verifikon User-in nga DB. Arsyeja: lockout-i duhet të hyjë në fuqi menjëherë, jo pas 15 minutash kur skadon JWT.

Refresh Token Rotation

Çdo refresh lëshon token të ri dhe revoke-on atë të vjetrin. Standard industry kundër vjedhjes së refresh tokens.

Kontrolli me dy nivele për Staff

roleMiddleware lejon hyrje → Controller bën kontroll shtesë: a është vetja, a po ndryshon is_active. Lejon granularitet të lartë pa middleware shtesë.
